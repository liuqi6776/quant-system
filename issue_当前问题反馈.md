# 当前问题反馈

## 1. 致命问题：训练标签时点泄露 (已修复)
* **问题描述**: 在原先 the 训练管道中，训练集截止日期使用的是 `df["trade_date"] < date` (即包含 `d-1` 交易日)。然而，对于 `d-1` 交易日的样本，其目标标签（预测未来 5 个交易日的涨幅）需要使用从 `d` 开盘价到 `d+5` 开盘价的收益率。这意味着在 `d-1` 时刻，我们需要使用未来 `d` 到 `d+5` 的数据来打标签，从而在训练集里引入了未来函数。
* **修复方案**: 
  1. 将训练集的截止日期严格限制为 `d-7` (即 `cutoff_idx = all_dates.index(date) - 7`)，确保训练集中最后一个样本的标签在其特征产生时点（`d-7` 收盘）至 `d-1` 收盘时已经完全实现，杜绝任何未来数据泄露。
  2. 这样做也自然对重叠持有期做出了 purging / embargo 隔离，保证模型训练의 严谨性。

## 2. 致命问题：`bias_5` / `bias_20` 跨股票污染 (已修复)
* **问题描述**: 分子在计算偏离度时使用了 `groupby("ts_code")`，但分母在计算滚动均值 `df["close"].rolling(w).mean()` 时没有进行 `groupby`。这导致分母在多股票面板数据中混合了不同股票在时序上的收盘价，导致特征计算完全错误且跨股票污染。
* **修复方案**: 将分母计算均值也改为 `groupby("ts_code")`，使用 `g.transform(lambda x: x.rolling(w).mean())` 确保严格的个股时序滚动计算。

## 3. 其他重大风险与实盘不可行警告 (待验证/提示)
* **高换手率与交易成本敏感**: 策略每周全仓换手（3只股票），相当于每周 100% 换手率。A股的印花税（千分之零点五，卖出收取）、佣金及 bid-ask 滑点对这种高频换手极其敏感。在考虑实际交易摩擦后，该策略的 standalone 表现退化为负收益（CAGR -12.16%），在实盘中完全不可行。
* **潜在幸存者偏差**: 股票池未严格剔除历史退市及指数成分股变动，可能存在幸存者偏差。
* **PE/PB 公告日对齐验证**: 需要进一步核实底层数据库中的 PE/PB 历史数据是按照公告日（Point-in-Time）对齐还是财报期末日对齐。如果是后者，亦会引入严重的未来函数泄露。
